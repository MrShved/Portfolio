# Анализ продуктовой аналитики

**Анализ продуктовой аналитики** (Analytics) — это количественный метод исследования, при котором анализируются данные о поведении пользователей в продукте.

## Когда применяется

- **Пост-запуск**: анализ поведения после выхода продукта
- **Оптимизация**: выявление проблемных зон в пользовательском пути
- **Валидация гипотез**: проверка предположений данными

## Цели исследования

✅ **Понимание поведения** — как пользователи взаимодействуют с продуктом  
✅ **Выявление паттернов** — обнаружение общих тенденций  
✅ **Оптимизация конверсии** — улучшение ключевых метрик  
✅ **Сегментация пользователей** — группировка по поведению

## Преимущества

✅ **Объективность** — данные, а не мнения  
✅ **Масштабируемость** — можно анализировать миллионы событий  
✅ **Автоматизация** — сбор данных автоматизирован  
✅ **Непрерывность** — можно отслеживать изменения во времени

## Недостатки

❌ **Поверхностность** — показывает "что", но не "почему"  
❌ **Требует данных** — нужно достаточное количество событий  
❌ **Сложность настройки** — требует правильной установки трекеров  
❌ **Контекстная ограниченность** — не видит внешние факторы

## Процесс проведения

1. **Аудит аналитики** — проверка установленных инструментов
2. **Определение метрик** — выбор ключевых показателей
3. **Настройка событий** — регистрация важных действий
4. **Сбор данных** — накопление достаточного объёма
5. **Анализ паттернов** — выявление тенденций
6. **Формулировка выводов** — интерпретация данных
7. **Рекомендации** — конкретные улучшения для продукта

## Типичные метрики

| Метрика | Описание |
|---------|----------|
| DAU/MAU | Дневная/месячная активная аудитория |
| Retention Rate | Процент возвращающихся пользователей |
| Churn Rate | Процент оттока пользователей |
| Conversion Rate | Процент выполнивших целевое действие |
| Funnel Analysis | Анализ прохождения воронки конверсии |

## Инструменты

Популярные платформы для аналитики:

- **Google Analytics** — веб-аналитика с глубокими возможностями
- **Mixpanel** — событийная аналитика для продуктов
- **Amplitude** — поведенческая аналитика с визуализацией путей
- **Heap** — автоматическая запись всех событий
- **Segment** — централизованный сбор событий

## Стоимость и сроки

- **Минимум**: 7 дней (для анализа существующих данных)
- **Максимум**: 30+ дней (для накопления достаточного объёма данных)
- **Доступ**: own (требуется свой продукт с установленной аналитикой)
- **Бюджет**: mid/high (зависит от инструмента и объёма данных)

## Сложность

**Medium-High** — требует знаний аналитики, настройки инструментов и интерпретации данных.

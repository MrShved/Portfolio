# Юзабилити-тестирование

**Юзабилити-тестирование** (Usability Testing) — это качественный метод исследования, при котором наблюдатели за тем, как реальные пользователи взаимодействуют с продуктом для выполнения конкретных задач.

## Когда применяется

- **Валидация**: проверка гипотез о поведении пользователей
- **Пост-запуск**: поиск проблем после выхода продукта
- **Итеративная разработка**: улучшение UX в процессе проектирования

## Цели исследования

✅ **Выявление барьеров** — найти точки трения в пользовательском пути  
✅ **Валидация дизайна** — проверить, понятны ли интерфейсы пользователям  
✅ **Понимание ментальных моделей** — узнать, как пользователи мыслят  
✅ **Приоритизация улучшений** — определить самые важные проблемы для исправления

## Преимущества

✅ **Глубокое понимание** — видите реальные действия и эмоции пользователей  
✅ **Раннее обнаружение проблем** — находите ошибки до разработки  
✅ **Конкретные доказательства** — факты вместо мнений  
✅ **Универсальность** — применим к любому продукту (веб, мобильное приложение, IoT)

## Недостатки

❌ **Требует модераторов** — нужна подготовка и проведение сессий  
❌ **Ограниченная масштабируемость** — обычно тестируется 5-8 пользователей  
❌ **Субъективность** — интерпретация наблюдений зависит от модератора  
❌ **Время на анализ** — нужно время для кодирования и анализа данных

## Процесс проведения

1. **Подготовка сценариев** — разработка задач для тестирования
2. **Рекрутинг участников** — подбор пользователей по критериям
3. **Настройка оборудования** — подготовка инструментов записи
4. **Проведение сессий** — наблюдение и модерация в реальном времени
5. **Кодирование данных** — категоризация наблюдений
6. **Анализ результатов** — выявление паттернов и инсайтов
7. **Формулировка рекомендаций** — конкретные улучшения для продукта

## Методология

- **Модерированные сессии** — исследователь ведёт тест в реальном времени
- **Немодерированные сессии** — автоматические записи без участия исследователя
- **Think aloud** — пользователи говорят о своих мыслях вслух
- **Критические инциденты** — фиксация моментов, когда пользователь застревает

## Типичные метрики

| Метрика | Описание |
|---------|----------|
| Task Success Rate | Процент успешно выполненных задач |
| Time on Task | Время на выполнение задачи |
| Error Rate | Количество ошибок при выполнении |
| SUS Score | Общий балл юзабилити (0-100) |

## Стоимость и сроки

- **Минимум**: 3 дня (для быстрого теста с панелью пользователей)
- **Максимум**: 14 дней (для глубокого исследования с рекрутингом)
- **Доступ**: own/panel (собственная панель или внешняя)
- **Бюджет**: low/mid (зависит от размера панели и количества сессий)

## Сложность

**Medium** — требует навыков модерации, подготовки сценариев и анализа качественных данных.
